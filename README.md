# JournaLog

## 日々の積み上げを可視化し、継続を支援するジャーナルアプリ

## 背景・課題
- 自己啓発系Youtuberである-ジョージ-さんの影響で、ジャーナルを書く習慣を始めた
- ジャーナルでは、目標やその日の行動、振り返りをノートに記録していた
- しかし、ノートに手書きで書く作業が次第に面倒に感じるようになった
- 電車などの移動中は混雑しており、ノートを開いて記録することが難しかった
- また、ノートは使い切るたびに買い直す必要があり、過去の記録を振り返りづらいという課題があった
- これらの記録をPCやスマートフォンから、いつでもどこでも入力・確認できたら便利だと感じた

## 解決策 / コンセプト
ノートの代わりに、Goal・Todo・スケジュール・振り返りの4つの要素を1つのJournalとして管理できるアプリを作成しました。
この4つを組み合わせることで、「目標 → 行動 → 振り返り」という流れを自然に作れる設計にしています。
これにより、ジャーナル習慣をPCやスマートフォンから無理なく継続できるようにしました。

## 主な機能（Features）

- **Goal（目標）管理**
  - 日々の目標を設定、編集、削除できる
  - 達成状況を確認して振り返りに活用可能

- **Todo（やること）管理**
  - 目標に紐づいたやることを作成、編集、完了管理できる
  - 進捗状況を可視化して効率的にタスクを管理

- **スケジュール管理**
  - 日付ごとにタスクやイベントを登録、編集、削除できる
  - 一覧表示で今日やることをすぐ確認

- **振り返り（Reflection）**
  - 1日の行動や達成度を簡単に記録できる
  - 過去の振り返りを見返して自己改善に活用可能

- **共通機能**
  - 認証機能（ログイン / ログアウト）
  - デバイスを問わずPC・スマホからアクセス可能

- **認証・表示制御**
  - ログインしたユーザーのみ、自分のGoal・Todo・スケジュール・振り返りを閲覧・編集可能
  - 他ユーザーのデータは表示されず、プライバシーを確保

## 工夫した点（Highlights）

- **共通テンプレート設計**
  - Goal / Todo / スケジュール(Schedule) / 振り返り(Reflection) のCreate・Update・Deleteで共通化できるbaseテンプレートを作成、
  - 複雑なファイル構成にならないよう意識しました。

- **UX向上の工夫**
  - JavaScriptを活用して一部アニメーションを導入
  - 一度に複数件のCreateを可能にする操作性を実現
  - AIのコーディング支援を活用しつつ、ユーザー視点で使いやすさを意識
